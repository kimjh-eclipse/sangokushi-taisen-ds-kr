# San Goku Shi Taisen (J) 한국어화 작업 기록

최종 업데이트: 2026-07-06

---

## 1. 개요

- **대상 ROM**: `San Goku Shi Taisen (J).nds` (삼국지대전 DS, SEGA/Namco, 게임코드 **A3DJ**, 128MB)
- **산출물**: `San Goku Shi Taisen (K).nds` (한국어판, 폴더 루트)
- **배포 패치**: `SGS_kr_patch_v20260706.xdelta` (~323KB) — 원본 J에 적용 시 K와 **MD5 완전 일치** 검증됨
- **명명 주의**: `SGS` = 삼국대전(본작), `SGST` = 삼국대전 **천(Ten)** (전작, 별개 게임). 폴더의 `SGST_kr_patch_v20260703/05.xdelta`는 전작 천 패치.
- **번역 전략**: 전작 천(K) 패치와 동일하게 **폰트의 SJIS 한자 슬롯을 한글로 재매핑**. 텍스트는 그 슬롯 코드로 재인코딩.

### 전작 천(J)과의 차이 (중요)
전작 `천(J)`(코드 CNTJ, 268MB)와 **포맷이 완전히 다른 별개 게임**이다. 천의 NFP1.0/ALPHA-UNIT/2CPK 지식은 통하지 않아 포맷을 처음부터 전부 재분석했다.

## 5. arm9/overlay 추출 노하우 (재현 핵심)

arm9는 코드+데이터 혼재라 텍스트만 엄격 추출해야 한다. 게임 텍스트가 **여러 형태·중복**으로 흩어져 있어, 화면별 미번역 지적마다 `arm9text.py`의 추출 조건을 단계적으로 넓혔다:

1. `extract` — 단줄(가나 포함, NUL종단)
2. `extract_ml` — **NUL구분 + 개행(\n) 다줄 통짜** (세력·모드 설명 등)
3. **UI마커(：％「」0-9%)+한자2** — 한자전용 UI 텍스트
4. **전각쉼표 `、` 등 마커 추가** — 스테이지명(反董卓軍、結成 등)
5. **순수 한자 3+ 허용** — 계급명·부대명·基本説明·戦闘準備
6. `is_text_char`를 **cp932 인쇄가능문자**로 완화 — 그리스ε·로마숫자Ⅰ·원문자① 등 기종의존문자 포함(汜水関=데이터상 "ε水関")

- **치환**: `apply()` — 번역 바이트 ≤ 원문 바이트 + NUL 패딩. `allow_nl=True`로 개행 유지.
- **중복 주의**: 같은 성격 텍스트가 여러 벌 존재(세력설명·계략설명 등). 화면이 **실제 쓰는 off**를 찾아야 한다.
- **스킵 대상**: 순수 히라가나 요미가나·가나 폰트표는 재매핑 안 되므로(원문 유지) 번역 불필요.

## 7. DeSmuME 자동화 노하우 (`emu.py`, `drive.py`)

- **터치**: 물리 `SetCursorPos`+`mouse_event`(AttachThreadInput 포커스 후) + **데스크톱 ImageGrab 좌표**. PostMessage/SendInput 합성입력은 터치 안 먹음.
- **스크린샷**: DeSmuME QuickScreenshot(WM_COMMAND 130 → Screenshots폴더)는 프레임버퍼(256×384 세로회전본). 데스크톱 ImageGrab은 창 렌더 정상표시. (PrintWindow는 게임화면이 검게 나옴)
- **함정 ①**: DeSmuME는 ROM 파일을 덮어써도 **자동 리로드 안 함** → 재빌드 후 File>Close/Reset 후 재로드 필수. (이 때문에 "안 됐다" 오인 사례 다수)
- **함정 ②**: 외부 디스플레이 분리 등으로 창이 화면 밖(off-screen)이면 캡처 회색 → `MoveWindow`로 primary 안으로.
- 이름입력 가나 터치는 화면배치 변동으로 불안정(알파벳 모드가 좌표 잡기 쉬움).

## 7.5 그래픽 식자 진행 (2026-07-06 착수, 사용자 요청: 버튼→메뉴→카드 전부)

### SCM 그래픽 포맷 (해독 완료)
- **MC 그래픽 = MC헤더(0x20B) + [팔레트 섹션][gfx 섹션]**. 헤더 u16[3]=폭, u16[4]=높이, u16[6]=팔레트 색수, u32@0x10=gfx섹션 해제크기.
- 팔레트: **raw**(색수×2B, BGR555, 0x20부터) 또는 LZ10. 판별: 0x20+색수*2 위치가 0x10(LZ10)이면 raw.
- gfx 섹션: LZ10(BIOS 헤더포함 `10 size24`) → 해제하면 **"CNCP" 스프라이트 셀 뱅크**.
- 폰트만 예외(허프만+LZ10 체인, `mc.py`). MC 저장방식 분류 스크립트 결과: LZ10 1572, 팔레트raw형(0x1f 등) 2163+488+…, 허프만 183, RAW 41.
- 도구: **`scmimg.py`**(MC그래픽 decode: 팔레트+gfx, `lz10_dec_track` 소비량 추적), **`cncp.py`**(CNCP 파서/셀 렌더러).

### CNCP 포맷 (해독 완료)
`'CNCP' ver | cellTblOfs,cellCnt | animTblOfs,animCnt | endOfs,0 | 0x20~ 4bpp타일gfx | cellTbl(16B×N=[oamListPtr,oamCnt,gfxBase=0x20,?]) | oam item 12B=attr0,attr1,attr2,pad,seq,pad`
- OAM 속성은 **표준 NDS OAM**: attr0(Y8,shape14-15), attr1(X9,hflip12,vflip13,size14-15), attr2(tile0-9,pal12-15). 타일 1D 매핑, 4bpp, 색0 투명, 서브팔레트=pal×16.
- `cncp.py render_all()`로 셀 시트 PNG 검증 완료(DCHOOSE=덱선택 버튼 決定/デッキエディット/名前変更/破棄 + 비활성 + 프레임, 18셀 완벽 렌더).
- **DCHOOSE.SCM = Deck Choose** 화면 버튼(손잡이 はい/いいえ 아님 — 그 파일은 아직 미특정).
- EX_NO.SCM = 숫자(0-9) 스프라이트.

### 그래픽 파이프라인 (기반 완성 — `scmimg.py`/`cncp.py`/`scmenc.py`)
- **재인코더 `scmenc.py rebuild(orig_scm, new_gfx)`**: MC헤더+팔레트섹션 원본유지 + 편집 gfx를 LZ10 재압축(ndspy.lz10.compress) + 0x10필드(gfx해제크기) 갱신. **무수정 왕복 gfx 일치 검증 완료**(BTNALL/DCHOOSE/EX_NO). 크기 약간 증가(LZ10 차이)하나 NFP repack이 오프셋 재계산하므로 무관.
- **はい/いいえ 버튼 = BTNALL.SCM** 확정. cell6=はい(tile120~,pal0 빨강), cell12=いいえ(tile240~,pal2 파랑). 그 외 BTNALL 48셀: 決定/計略説明/戻る/破棄 등 공통버튼(세력색 5종). DCHOOSE=덱편집버튼, AP.SCM=42셀(계급/작위 아이콘 등).

### BTNALL.SCM 전 버튼 식자 완료 (2026-07-07, 인게임 검증 완료)
- **`61_btnall_all.py`** (파일럿 `60_btnall_kr.py`의 일반화): 12버튼×3상태 + 회색 12종 전부 한글 식자.
  - 결정(0-2, 3-5 중복버튼), 예(6-8), 뒤로(9-11), 아니오(12-14), 전환(15-17), 덱편성(18-20), 이름변경(21-23, 32x48), 파기(24-26), 교환(36-38), 계략설명(40-42, 32x48), 삭제(44-46) + 회색 27-35/39/43/47.
- **글자/구슬 분리 원리 (핵심 노하우)**:
  1. **글자는 상태(3연속 셀) 간 불변, 구슬(오브)은 상태 간 변함** → 상태간 다른 픽셀 = 100% 구슬.
  2. 같은 팔레트·상태의 다른 버튼과 만장일치 = 안정 구슬(그림자·외곽).
  3. 남은 픽셀 = 글자 → 같은 팔레트 타 버튼 중 "그 픽셀이 구슬(상태변화 있음)인 버튼"에서 값 차용.
  - 실패했던 접근: 단순 다수결(서로 다른 버튼의 글자 외곽선 인덱스가 우연히 일치해 잔해 남음), 회색 구슬 기반 tmask(회색 셀들은 구슬 무늬가 서로 미세하게 달라 오염).
- **회색(단일상태) 버튼**: 컬러 counterpart와 글자 모양 100% 일치(자카드 1.00 매칭: 0↔27, 3↔28, 6↔29, 9↔30, 12↔31, 15↔32, 18↔33, 21↔34, 24↔35, 36↔39, 40↔43, 44↔47) → 컬러 분석에서 얻은 글자마스크 위치를 지우고 타 회색버튼 값으로 치환.
- **한글 렌더**: 굴림(gulim.ttc) 13px(2자 이하)/11px(3자+), 문자열 가로 렌더 → **90° 시계방향 회전**(글자가 타일 공간에서 세로쓰기+회전 저장돼 있음, 게임기를 세로로 들고 플레이) → 채움=최명색 인덱스(F), 외곽선=최암색(팔레트별 5/6/7/4), MaxFilter(3) 팽창 1px.

- **인게임 검증**: 손잡이 확인 "예/아니오" ✔, 이름 확인 다이얼로그 "예/아니오" ✔, 스타터덱 입수 "결정" ✔ (DeSmuME 터치 자동화로 첫부팅 흐름 주파).

### 판형(plate) 버튼/배너 식자 (2026-07-07, `63_plates_kr.py` — 미리보기 검증, 인게임 위치 미확인)
